# 사진 갤러리 모달 설계

## 목적
가게 대표사진(`photo`)은 그대로 두고, 여러 장 사진을 넘겨볼 수 있는 라이트박스 모달 추가. 나중에 사진 추가 시 코드 수정 없이 데이터만 넣으면 되게.

## 데이터 모델
- 기존 `photo` 필드 유지 (대표사진, 마킹 작업 대상 그대로).
- 신규 옵션 필드 `extraPhotos: string[]` 추가. 없으면 갤러리는 대표사진 1장짜리로 동작 (기존과 동일).
- 모달에 넘길 사진 배열: `[item.photo, ...(item.extraPhotos || [])].filter(Boolean)`.

## UI / 인터랙션
- 카드 썸네일 우측하단 확대아이콘 클릭 → 모달 오픈 (2026-07-11 리버트된 라이트박스 커밋(`fc64562`) 패턴 재사용).
- 사진 1장: 확대뷰만 (화살표/점 없음).
- 2장 이상: 좌우 화살표 버튼 + 하단 점(dots) 인디케이터 + 모바일 스와이프(터치 좌우 드래그).
- 닫기: X 버튼 / 오버레이 클릭 / ESC — 기존과 동일.
- 확대아이콘은 모바일에서도 노출 (그리드 탭펼침 `initTileToggle`과 겹치지 않게 이벤트 stopPropagation 처리).

## 구현 위치
- `card()` / `magazineCard()`: 확대아이콘 버튼에 `data-photos` 속성으로 JSON 배열 부여 (또는 `data-photo-name`으로 DATA 재조회 후 배열 구성 — 구현 시 결정).
- 신규 함수 `initPhotoModal()`: `#results` 이벤트위임으로 확대아이콘 클릭 감지, 모달 열기/화살표/점/스와이프/닫기 전부 처리.
- 기존 필터/정렬/render() 로직 무변경.

## 테스트 / 검증
- 사진 1장짜리 가게: 기존과 동일하게 확대뷰만 뜨는지 확인.
- `extraPhotos` 2장 이상 넣은 가게 하나로 화살표/점/스와이프 동작 확인.
- 모바일 폭(≤620px)에서 확대아이콘 클릭 시 타일펼침 토글이 같이 발동하지 않는지 확인.
- ESC/오버레이클릭/X 버튼 전부 닫히는지 확인.

## 범위 밖
- 그리드 카드 자체에서의 스와이프 (모달 안에서만 지원).
- 사진 업로드/편집 UI (URL만 데이터에 추가하는 방식 유지).
